# 知识库 MCP + Skill 安装说明

本文档面向 AI 助手（Claude Code / Codex 等）。用户会把本文档丢给你，让你帮他在本机接入知识库系统。**请你按本文档的步骤逐项执行**，不要要求用户手动操作。

执行完每一步都要校验结果，失败立即停下来告诉用户原因，不要继续往下走。

---

## 0. 角色与目标

知识库系统已经在本机以直装版形式安装并运行，监听 `127.0.0.1:18000`（端口可能被用户改过）。你的任务是：

1. 把 **MCP server**（`kb-mcp-proxy.py`）注册到当前 AI 客户端的 MCP 配置文件里。
2. 把 **Skill 文件**（`SKILL.md`）复制到当前 AI 客户端的 skills 目录。
3. （仅 Claude Code 需要）把若干安全工具加入权限白名单，避免每次调用都问用户。
4. 验证安装成功。

本目录（`<install_dir>/agent-integration/`）下你会用到的文件：

| 文件 | 用途 |
|---|---|
| `kb-mcp-proxy.py` | MCP server 入口，**不要修改内容**，注册到 MCP 配置时用绝对路径引用 |
| `SKILL.md` | Skill 主干，**整文件**复制到客户端 skills 目录 |
| `安装说明.md` | 本文档 |

---

## 1. 检测前置条件

先确认下面三件事，全部满足才继续：

1. **知识库服务正在运行**
   ```
   curl http://127.0.0.1:18000/health
   ```
   预期：`{"status":"ok"}`。如果换了端口，先看 `<install_dir>/config/config.toml` 的 `[server].port`。如果 curl 不通，让用户先打开托盘 / 菜单栏 App 启动服务。

2. **Python 3.10+ 可用**
   ```
   python3 --version   # macOS / Linux
   python --version    # Windows（多数情况下也叫 python）
   ```
   预期：`3.10` 或更高。MCP server 跑在用户系统 Python 上，不依赖 venv。

3. **`mcp` 与 `httpx` 包已安装**
   ```
   python3 -c "import mcp, httpx"
   ```
   报 ImportError 就让用户执行：
   ```
   python3 -m pip install --user mcp httpx
   ```
   装完再重试。

## 2. 解析本机绝对路径

后续步骤要把 `kb-mcp-proxy.py` 的绝对路径写进配置文件，路径必须**用户家目录展开 + 绝对路径**，不能用 `~` 或相对路径（不同客户端展开行为不一致）。

### 2.1 本目录绝对路径

把本文档所在目录的绝对路径记下来。以下示例假设：

| 平台 | 典型安装目录 |
|---|---|
| macOS | `/Applications/KnowledgeBase` |
| Windows | `C:\Users\<用户名>\AppData\Local\KnowledgeBase`（即 `%LocalAppData%\KnowledgeBase`） |
| Linux / 开发模式 | 仓库根目录 |

`<agent_integration_dir>` 即上述目录下的 `agent-integration/` 子目录。

### 2.2 MCP server 入口绝对路径

`<mcp_proxy_path>` = `<agent_integration_dir>/kb-mcp-proxy.py`

### 2.3 用户家目录

| 平台 | 家目录 |
|---|---|
| macOS / Linux | `$HOME`，例 `/Users/alice` |
| Windows | `%USERPROFILE%`，例 `C:\Users\alice` |

后续 `<home>` 占位即家目录绝对路径。

## 3. 注册 MCP server

根据用户的 AI 客户端选择对应配置文件。如果用户没说，默认两个都装。

### 3.1 Claude Code

配置文件：`<home>/.claude/.mcp.json`（不存在则新建）。

**目标结构**（其他已有 MCP 条目保留，只追加 `knowledge-base-system`）：

```json
{
  "mcpServers": {
    "knowledge-base-system": {
      "command": "python3",
      "args": ["<mcp_proxy_path>"],
      "env": {}
    }
  }
}
```

操作要点：
- 已存在则用 JSON 合并方式追加 key，**不要覆盖整个文件**。
- 已有同名条目（`knowledge-base-system`）：先把旧配置备份到 `<home>/.claude/.mcp.json.backup.<时间戳>`，再覆盖。
- Windows 上 `command` 用 `python`（不是 `python3`），路径里的反斜杠在 JSON 里要双写：`"C:\\Users\\alice\\..."`。

### 3.2 Codex

配置文件：`<home>/.codex/config.toml`（不存在则新建）。

**追加段落**：

```toml
[mcp_servers.knowledge-base-system]
command = "python3"
args = ["<mcp_proxy_path>"]
```

操作要点：
- TOML 不是结构化合并友好，**追加在文件末尾**即可。
- 已有同名段落：先把整个文件备份到 `<home>/.codex/config.toml.backup.<时间戳>`，再用新段落替换。
- Windows 上 `command = "python"`；路径里的反斜杠要在 TOML 双引号字符串内转义：`"C:\\Users\\alice\\..."`。

## 4. 安装 Skill 文件

把本目录的 `SKILL.md` 整文件复制到对应客户端的 skills 目录。如果旧文件存在，先备份再覆盖。

### 4.1 Claude Code

目标路径：`<home>/.claude/skills/knowledge-base-first/SKILL.md`

操作：
1. 创建目录 `<home>/.claude/skills/knowledge-base-first/`（如不存在）。
2. 若已存在旧的 `SKILL.md`：备份到 `SKILL.md.backup.<时间戳>`。
3. 把 `<agent_integration_dir>/SKILL.md` 复制过去。

### 4.2 Codex

目标路径：`<home>/.codex/skills/knowledge-base-first/SKILL.md`

操作步骤同上，把路径里的 `.claude` 换成 `.codex`。

## 5. 权限白名单（仅 Claude Code）

Claude Code 的 `<home>/.claude/settings.json` 控制工具调用是否需要每次确认。把以下安全工具加入白名单可以让用户少被打断。

**目标 permissions 片段**（已存在则按 JSON 合并）：

```json
{
  "permissions": {
    "allow": [
      "mcp__knowledge-base-system__search_knowledge",
      "mcp__knowledge-base-system__get_knowledge_item",
      "mcp__knowledge-base-system__upsert_knowledge",
      "mcp__knowledge-base-system__import_incremental_knowledge",
      "mcp__knowledge-base-system__export_knowledge_package"
    ]
  }
}
```

操作要点：
- 文件不存在则新建一个含上面结构的 JSON。
- 文件存在则把这 5 个字符串追加进 `permissions.allow` 数组（去重）。
- 备份原文件到 `settings.json.backup.<时间戳>`。
- **不要**把高风险工具（`import_knowledge_package` / `clear_knowledge_base` / `cleanup_expired_knowledge`）加进白名单，它们必须每次让用户确认。

Codex 当前没有等价的全局权限白名单机制，跳过本步骤。

## 6. 验证安装

按下面顺序逐项检查，**任何一步失败就停下报错给用户**。

### 6.1 验证 MCP 注册

Claude Code：
```
claude mcp list
```
预期输出包含 `knowledge-base-system`。

Codex：
```
codex mcp list
```
预期同上。

### 6.2 验证 Skill 文件

```
ls <home>/.claude/skills/knowledge-base-first/SKILL.md
ls <home>/.codex/skills/knowledge-base-first/SKILL.md
```
Windows 用 `dir`。

### 6.3 验证服务连通

让用户在客户端会话里跑一个真实检索任务，例如：

> 在知识库里搜一下 "MCP" 相关条目。

观察客户端是否调用 `mcp__knowledge-base-system__search_knowledge` 并返回结果。如果命中应在末尾看到 `KB Trace: trace_id=...; knowledge_item_id=...`。

## 7. 端口变更后

如果用户后续在控制台改了 `service_port`：

- MCP 代理（`kb-mcp-proxy.py`）会自动读取 `<install_dir>/config/config.toml`，**无需重新安装**。
- 重启 Claude Code / Codex 会话即可让 MCP server 重新读取新端口。

## 8. 卸载

如用户要求卸载：

1. 从 `<home>/.claude/.mcp.json` 的 `mcpServers` 中删除 `knowledge-base-system` 条目。
2. 从 `<home>/.codex/config.toml` 中删除 `[mcp_servers.knowledge-base-system]` 段。
3. 删除 `<home>/.claude/skills/knowledge-base-first/` 与 `<home>/.codex/skills/knowledge-base-first/` 整个目录。
4. 从 `<home>/.claude/settings.json` 的 `permissions.allow` 中去掉 `mcp__knowledge-base-system__*` 条目。

每一步前先备份，方便用户回退。

## 9. 操作系统差异速查

| 概念 | macOS / Linux | Windows |
|---|---|---|
| 路径分隔符 | `/` | `\`，JSON / TOML 字符串中要写成 `\\` |
| Python 命令 | `python3` | 通常是 `python`（少数环境也叫 `python3`） |
| 用户家目录 | `$HOME` / `~` | `%USERPROFILE%`（避免在 JSON / TOML 中用变量，全部展开为绝对路径） |
| 列目录 | `ls` | `dir` |
| 默认安装目录 | `/Applications/KnowledgeBase` | `%LocalAppData%\KnowledgeBase` |
| 配置文件位置 | `~/.claude/` `~/.codex/` | `%USERPROFILE%\.claude\` `%USERPROFILE%\.codex\` |

## 10. 异常处理速查

| 现象 | 处理 |
|---|---|
| `curl /health` 不通 | 让用户启动托盘 / 菜单栏 App 的「启动知识库」；再不通查 `<install_dir>/logs/api.err.log` |
| `python3 -c "import mcp"` 报 ImportError | `python3 -m pip install --user mcp httpx` |
| `claude mcp list` 不显示 | 检查 JSON 语法是否合法（多余逗号 / 转义错误）；备份恢复 `.mcp.json.backup.<时间戳>` |
| 调用工具时报 `kb-mcp-proxy` 启动失败 | 用 `python3 <mcp_proxy_path>` 单独跑一次看 stderr；常见原因是 Python 版本太老或 `mcp` 没装好 |
| 端口冲突 | 让用户在控制台「系统配置」改 `service_port`，重启服务，重启 AI 客户端 |
