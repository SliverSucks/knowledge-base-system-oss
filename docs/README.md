## 维护规则

1. 新文档必须标注受众：`用户` 或 `开发/运维`。
2. 文档更新后需同步更新本导航，避免入口失效。
3. 平台差异（Windows/Mac）只写安装与启动差异，不分叉核心业务说明。
4. 文档是现状说明，不写历史变更、未实现功能与里程碑叙事。
