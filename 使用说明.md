# 知识库 (Knowledge Base) 使用说明

本文档覆盖 macOS 与 Windows 直装版。两端核心功能一致，仅安装路径与启动方式有差异；**右键托盘菜单的少数管理功能（增量导入 / 清空 / 清理过期）当前仅 macOS 提供**，Windows 端会在后续版本补齐。

---

## 1. 安装

**Windows**

双击安装程序 `KnowledgeBase-Setup-x.y.z.exe`，按向导默认设置即可。

- 安装目录：`%LocalAppData%\KnowledgeBase`
- 完成后开始菜单 → 知识库 → 启动托盘程序

**macOS**

双击 `KnowledgeBase-mac-direct-x.y.z.dmg`，再双击 `Install.command`。

- 安装目录：`/Applications/KnowledgeBase`
- 完成后自动打开菜单栏 App（`KnowledgeBaseMenuBar.app`）

---

## 2. 启动 / 停止 / 打开界面

托盘 / 菜单栏图标点击展开，两个平台功能一致：

| 菜单项 | 行为 |
|---|---|
| 状态指示（不可点击） | 当前服务状态 |
| 启动知识库 | 启动后台服务（约 10~20 秒就绪） |
| 停止知识库 | 停止后台服务 |
| 打开知识库工作台 | 浏览器打开 `http://127.0.0.1:18000/console` |
| 打开系统配置 / 设置 | 浏览器打开 `http://127.0.0.1:18000/settings` |
| 打开 API 文档 | 浏览器打开 `http://127.0.0.1:18000/docs` |
| 查看状态 / 诊断状态 | 弹窗显示端口 / PID / 健康检查结果 |
| 知识库管理（子菜单） | 见下方平台对照 |
| 退出 | 关闭托盘 / 菜单栏 App，同时停掉后台服务（两端一致） |

**「知识库管理」子菜单平台差异**：

| 子菜单项 | macOS | Windows |
|---|:---:|:---:|
| 导入知识包（.tar.gz 备份 / .md / .txt / .docx / .pdf） | ✅ | ✅ |
| 导出知识包（流式 .tar.gz） | ✅ | ✅ |
| 打开导出目录 | — | ✅ |
| 增量导入（目录 → project/domain/type） | ✅ | 暂不支持 |
| 清空知识库 | ✅ | 暂不支持 |
| 清理过期知识 | ✅ | 暂不支持 |

> Windows 暂不支持的三项依赖 PostgreSQL/Qdrant 服务端模式的 bash 脚本工具链，与直装版（SQLite + 嵌入 Qdrant）的运行时不匹配。后续版本会补对应 HTTP API 后从托盘菜单接入。临时变通：导出全量包 → 编辑外部数据 → 导入覆盖。

图标状态：

- **运行中** = 服务正常
- **未运行** = 服务已停
- **进行中** = 启动 / 停止过程中

---

## 3. 修改端口

默认端口 18000。如有冲突，二选一：

**通过控制台（推荐）**

打开「系统配置」页 → 修改「服务端口」→ 保存 → 重启知识库。系统自动回写引导配置文件。

**直接改文件**

| 平台 | 配置文件位置 |
|---|---|
| Windows | `%LocalAppData%\KnowledgeBase\config\config.toml` |
| macOS | `/Applications/KnowledgeBase/config/config.toml` |

修改 `[server].port` 后从菜单「停止 → 启动」生效。

---

## 4. 数据位置

知识库的所有数据：

| 平台 | 数据目录 |
|---|---|
| Windows | `%LocalAppData%\KnowledgeBase\data\` |
| macOS | `/Applications/KnowledgeBase/data/` |

子目录：

- `knowledge.db` — SQLite 数据库（知识正文 + 配置）
- `qdrant_local/` — 向量索引（语义检索加速）
- `models/` — 内置 Embedding 模型权重（启用本地服务后才有）
- `embedding-service/venv/` — Embedding 服务独立 Python 环境（同上）

LLM / Embedding / Rerank 等业务配置存在 `knowledge.db` 里，不需要 `.env` 文件。通过控制台「系统配置」页修改即可。

## 4.5 启用语义检索（Embedding 服务）

不配置 Embedding 服务也能用，但只能用关键词检索（BM25）。要让 AI 理解"意思相近"而不只是关键词，需要启用 Embedding。

**首次启动后会自动跳转 `/setup` 引导页**，三选一：

### 选项 ①：本地内置（推荐）
适合：本机有 ≥ 4GB 可用内存，希望数据不出本机。
- 默认模型 BGE-M3，需下载约 2.3GB 权重
- 一键安装：选模型 → 点开始 → 等进度条跑完（约 5-15 分钟，取决于网速）
- 后续后台自动保活，无需手动维护

### 选项 ②：外部服务
适合：已经有 Azure OpenAI / 私有 embedding 部署。
- 填 base_url / api_key / model 三项即可
- ⚠️ 数据会发往外部 API，请确认合规

### 选项 ③：先跳过
适合：只想用关键词检索；可随时回 `/setup` 启用。

### 切换 / 重建索引

启用后可在「系统设置」→「Embedding 服务」分区：
- **切换模式** — 重新走 `/setup` 流程
- **重建索引** — 切换模型后需重建，按提示输入 `I-CONFIRM-OVERWRITE` 确认
  - 中途可点「中止并回滚」恢复旧索引
  - 大库重建期间检索自动降级到关键词

**v1.3 新增：托盘菜单一键重建（macOS）**

菜单栏 → 知识库管理 → **重建向量索引** → 弹原生确认框 → 点"开始重建" → 自动后台跑：
- 弹"已启动"通知后可关掉窗口干别的
- 跑完（约 1-3 分钟，~1000 chunk · CPU bge-m3 约 30-60 秒）自动弹"重建完成 · 处理 X/Y · 用时 N 秒"通知
- 失败弹错误通知带详细原因

### 常见情况

- **下载卡住** — 默认走 hf-mirror.com 镜像；失败会自动切到 huggingface.co
- **首次检索很慢** — 模型刚加载需要预热，看到「⏳ 模型加载中」banner 时等 30-60 秒即可
- **服务退出后没释放端口** — 重启托盘 App 即可，会自动清残留 PID + 端口
- **看似一直在加载（banner 不变绿）** — v1.3 起壳层自动自愈：若 infinity 进程真的健康（`/health` 返 200），下一轮 reconcile 会强制清除卡住的"加载中"状态，无需手动重启
- **切换 embedding 模式后什么都没发生** — v1.3 起切换会自动触发对应动作：进入本地 → 自动 install + 启动 infinity；离开本地 → 自动停 infinity 释放内存；改模型 → 自动 stop+install+start 串联

## 5. 数据备份

### 5.1 自动备份（升级 / 恢复时触发）

安装新版本时，安装器会**先**把旧目录复制到 auto-backup，**核心数据复制失败立刻中止安装、旧数据原封不动**；复制成功后才进入文件覆盖阶段。同时安装器会检测知识库进程是否在跑，在跑则提示用户先退出托盘再装（防 SQLite / Qdrant snapshot 不一致）。

**v1.3 起备份范围扩展到三个目录**（macOS APFS 用 clonefile 瞬时备份，几乎零空间开销）：

| 目录 | 失败处理 | 说明 |
|---|---|---|
| `data/` | 失败立即 abort | 知识库 SQLite + Qdrant 向量索引，核心数据不容丢失 |
| `models/`（v1.3 新增）| 失败仅警告 | 1-4 GB 模型权重，备份命中 → 升级后**跳过 4GB 下载**；失败后新版本走 `/setup` 重下 |
| `embedding-service/`（v1.3 新增）| 失败仅警告 | venv 含 infinity-emb 等依赖，备份命中 → 升级后**跳过 pip install**；失败后新版本重装 |

| 平台 | 自动备份位置 |
|---|---|
| Windows | `%LocalAppData%\KnowledgeBase\auto-backup\{时间戳}\`（含 `manifest.json`） |
| macOS | `~/Library/Application Support/KnowledgeBase/auto-backup/{时间戳}/`（含 `manifest.json`） |

快照永久保留，**建议每季度手动清理**一次旧目录（特别是 models/ 占 GB 级空间）。出问题时可通过托盘「知识库管理 → 导入知识包」选 `data\` 上层打包出来的 `.tar.gz` 恢复，或直接 cp data 回来。

### 5.2 手动备份

**方式 A：直接复制整个 `data/` 目录**到其他位置。建议先「停止知识库」再复制，避免数据不一致。

**方式 B：托盘 / 菜单栏 → 知识库管理 → 导出知识包**

生成一个 `.tar.gz` 包，含 `knowledge.db` + `qdrant_local` + 脱敏 manifest。

- Windows：保存到 `%LocalAppData%\KnowledgeBase\exports\kb-export-{时间戳}.tar.gz`，菜单里有「打开导出目录」直接定位
- macOS：保存到 `/Applications/KnowledgeBase/exports/kb-export-{时间戳}.tar.gz`

建议存到云盘 / 移动硬盘做异机备份。

> ⚠️ 备份包含 LLM / Embedding API key 等敏感凭证，**请勿上传公开仓库**或通过未加密渠道发送。

## 6. 开机自启（可选）

**Windows**：`Win + R` 输入 `shell:startup` → 把「知识库」快捷方式复制到打开的文件夹。

**macOS**：系统设置 → 通用 → 登录项 → 添加 → 选 `KnowledgeBaseMenuBar.app`。

---

## 7. 卸载

**Windows**：控制面板 → 程序和功能 → 知识库 → 卸载。

卸载不删除 `data\`。如需彻底清除，手动删 `%LocalAppData%\KnowledgeBase\`。

**macOS**：退出菜单栏 App，将 `/Applications/KnowledgeBase` 整个目录拖入废纸篓。

如需彻底清除 auto-backup：另删 `~/Library/Application Support/KnowledgeBase/`。

## 8. 接入 AI Agent（Claude Code / Codex）

让 Claude Code / Codex 在编码时自动检索知识库，需要做两件事：注册 MCP 工具 + 安装 Skill 行为规则。整个过程交给 AI 自己完成。

操作步骤：

1. 确保知识库服务正在运行（托盘 / 菜单栏图标显示运行中）
2. 打开一个 Claude Code 或 Codex 会话
3. 把以下一句话发给它：

   **macOS**：
   ```
   请按 /Applications/KnowledgeBase/agent-integration/安装说明.md 帮我接入知识库。
   ```

   **Windows**（把 `<用户名>` 换成你的实际用户名）：
   ```
   请按 C:\Users\<用户名>\AppData\Local\KnowledgeBase\agent-integration\安装说明.md 帮我接入知识库。
   ```

4. AI 会自动完成：
   - 检测前置条件（Python 3.10+、`mcp` 与 `httpx` 包是否已装）
   - 写入 MCP 配置文件
   - 复制 Skill 文件到客户端目录
   - 合并 Claude Code 权限白名单
   - 调用 `claude mcp list` / `codex mcp list` 验证

5. 装完后让 AI 跑一句「在知识库里搜一下 XXX」试试，能回答出结果即成功。

**前置条件**：

- Python 3.10 或更高版本必须已安装（`kb-mcp-proxy` 需要）
- 第一次使用让 AI 自动跑：`python3 -m pip install --user mcp httpx`

详细的接入步骤、操作系统差异、卸载方法都写在 `agent-integration/安装说明.md` 里，AI 助手会自己读。

## 9. 常见问题

**Q：托盘 / 菜单栏图标显示「启动超时」？**

A：后台服务启动约 10~20 秒；超时说明出现异常。点击「诊断状态 / 查看状态」看详情，或查看日志：

- Windows：`%LocalAppData%\KnowledgeBase\logs\`
- macOS：`/Applications/KnowledgeBase/logs/`

重点看 `api.err.log` 末尾。

**Q：端口冲突？**

A：见上方 §3 修改端口。

**Q：升级后配置或数据丢了？**

A：先看 `auto-backup/` 目录最新一份快照（见 §5 数据备份），通过托盘 / 菜单栏「知识库管理 → 导入知识包」选 `.tar.gz` 包恢复。

**Q：怎么让 Claude Code / Codex 用上知识库？**

A：见上方 §8 接入 AI Agent。
